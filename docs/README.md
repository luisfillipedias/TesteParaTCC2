# Documentacao

Esta pasta deve reunir os artefatos de documentacao de engenharia de software.

## Conteudo esperado

- `diagramas/`: casos de uso, classes, sequencia, implantacao, modelo de dados etc.
- `documentacao-software/`: PDF final da documentacao oficial.

## Recomendacoes

- Use nomes de arquivos claros, com versao e data quando necessario.
- Evite arquivos temporarios (exemplo: exportacoes intermediarias).
- Mantenha consistencia visual dos diagramas e nomenclatura.
